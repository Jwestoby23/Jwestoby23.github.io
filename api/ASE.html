Namespace ASE

Classes

ASECanvas

Concrete application canvas that implements both IappInterface and ISavableCanvas. Provides drawing primitives using GDI+ and maintains an internal bitmap for rendering.

AppCommandFactory

Factory that maps command names used by the ASE language to concrete BOOSE.ICommand implementations. The factory returns the appropriate command object given a canonical command type string.

Ase_Form

Main Windows Forms UI for the ASE application. Hosts the input textbox, execute/save buttons and a PictureBox used to display the canvas bitmap. Delegates execution and saving actions to ASE.Ase_Form.appBoose.

NoCommand

Placeholder command used for lines that should be ignored (for example comment markers). Implements the basic command lifecycle but performs no action during Compile or Execute.

appAdapter

Adapts an object that implements IappInterface so it can be used where an BOOSE.ICanvas is required. All calls are forwarded to the wrapped IappInterface instance.

appArray

Base implementation for array support in the ASE language. Provides compile-time parsing and runtime handling for both integer and real arrays, and helper routines used by concrete array commands (peek/poke).

Responsibilities:

  • Parse and validate array declarations (type, name, size) during Compile().
  • Allocate array storage on Execute().
  • Support reading and writing array elements through GetIntArray(int, int), GetRealArray(int, int), SetIntArray(int, int, int) and SetRealArray(double, int, int).
  • Provide parameter processing helpers (ProcessArrayParametersCompile(bool) and ProcessArrayParametersExecute(bool)) used by peek/poke derived commands.

Note: this class uses MessageBox calls to report user-visible errors to match the existing application's UX conventions.

appBoose

Facade and runtime coordinator for the ASE application. Manages the application canvas, command factory, parser and stored program. Exposes a minimal API used by the UI to run parsed programs and retrieve the canvas bitmap. This class implements a thread-safe lazy singleton via CreateInstance().

appInt

Represents an integer variable/command in the ASE language. The class can register itself with a BOOSE.StoredProgram and updates the program value when executed.

appParser

Application-specific parser that converts a single line of ASE language into an BOOSE.ICommand. Extends the generic BOOSE.Parser to add ASE-specific preprocessing, comment handling and assignment resolution where the left-hand side references an existing variable.

appPeek

Implements the "peek" array command which reads a value from an existing array variable and assigns it into another variable. Parsing of the textual parameters, index evaluation and array access are delegated to the helper logic implemented on appArray. This command reports user-visible problems using MessageBox calls to match the existing application's UX.

appPoke

Implements the "poke" array command which writes a value into an existing array variable. Uses the array helper logic provided by appArray to parse parameters at compile time and to perform the write at runtime.

appReal

Represents a real (double) variable in the ASE language. Registers itself with the active BOOSE.StoredProgram during compilation and updates the stored program value when executed. The class exposes a real property that holds the double backing value.

appStoredProgram

Application-specific BOOSE.StoredProgram implementation. Responsibilities:

  • Store and locate ASE variables.
  • Maintain a separate double backing for appReal variables and update both backing and textual value when UpdateVariable(string, double) is called.
  • Evaluate numeric expressions substituting real for real variables and BOOSE.Evaluation.Value for other variables before computing the result.
appWrite

Implements the "write" command which outputs text to the canvas. Execution order:

  • Attempt to evaluate the parameter as an expression via the program.
  • If the evaluation fails, treat the parameter as a variable name and write its value.
  • Otherwise treat the parameter as a literal text string and write it.

Interfaces

ISavableCanvas

Indicates that a canvas implementation can save its current bitmap to disk. Implementations should persist the visible canvas image to the provided file path.

IappInterface

Extended application canvas interface that exposes floating-point drawing primitives in addition to the standard BOOSE.ICanvas operations. Implementors are expected to provide both integer- and float-based circle/rectangle methods.
