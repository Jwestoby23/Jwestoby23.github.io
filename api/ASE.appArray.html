Class appArray

ASE.dll

Base implementation for array support in the ASE language. Provides compile-time parsing and runtime handling for both integer and real arrays, and helper routines used by concrete array commands (peek/poke).

Responsibilities:

  • Parse and validate array declarations (type, name, size) during Compile().
  • Allocate array storage on Execute().
  • Support reading and writing array elements through GetIntArray(int, int), GetRealArray(int, int), SetIntArray(int, int, int) and SetRealArray(double, int, int).
  • Provide parameter processing helpers (ProcessArrayParametersCompile(bool) and ProcessArrayParametersExecute(bool)) used by peek/poke derived commands.

Note: this class uses MessageBox calls to report user-visible errors to match the existing application's UX conventions.

ProcessArrayParametersCompile(bool)

Parse and prepare array operation parameters at compile time for peek/poke commands.

  • The peekOrPoke flag selects which side of an assignment to treat as the value.
  • Populates internal fields such as varName, rowS, columnS and pokeValue.
  • Reports parameter or variable resolution errors using MessageBox.
protected virtual void ProcessArrayParametersCompile(bool peekOrPoke)

Parameters

peekOrPoke bool

True when compiling a poke (write) command; false for peek (read).

ProcessArrayParametersExecute(bool)

Evaluate and perform peek/poke operations at execution time.

  • Resolves expressions for row/column/value where necessary.
  • Validates the parsed indices and converts the poke value to the array element type.
  • Performs read or write on the target array instance.
  • Reports errors using MessageBox for invalid indices, parse failures or unrecognised types.
protected virtual void ProcessArrayParametersExecute(bool peekOrPoke)

Parameters

peekOrPoke bool

True to perform a poke (write); false to perform a peek (read).
