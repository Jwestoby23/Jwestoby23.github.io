Class appPeek

Implements the "peek" array command which reads a value from an existing array variable and assigns it into another variable. Parsing of the textual parameters, index evaluation and array access are delegated to the helper logic implemented on appArray. This command reports user-visible problems using MessageBox calls to match the existing application's UX.

CheckParameters(string[])

Validate the parameter token count for a peek operation. Accepts 3 or 4 parameters in forms such as: [targetVar] = peek [array] [row] [targetVar] = peek [array] [row] [column] When the token count is invalid a MessageBox is shown (consistent with existing project behaviour).

Compile()

Compile-time processing for the peek command. Delegates parameter parsing to ProcessArrayParametersCompile(bool) with peekOrPoke = false. This prepares the command's internal fields (such as the array name and index expressions) for execution.

public override void Compile()

Execute()

Execute-time processing for the peek command.

  • Evaluates any index or value expressions via ProcessArrayParametersExecute(bool).
  • Retrieves the value from the target array and updates the destination variable on the program using UpdateVariable(string, double) (int values are promoted to double when required).
  • If the array type cannot be recognised a MessageBox is shown to report the problem.
